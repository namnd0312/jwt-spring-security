## Key Insights

- Hibernate DDL is `create-drop` so tables auto-created from `@Entity` classes; no manual SQL needed
- Java 8 constraint: use `java.util.Date` for expiry fields (not `java.time.Instant` which needs converters on older JPA)
- Existing entities use `@Data` (Lombok) and `@GeneratedValue(strategy = GenerationType.IDENTITY)`
- User entity has no `@Column` annotations currently (relies on Hibernate naming strategy for snake_case)
- RefreshToken uses `@OneToOne` to User (one active refresh token per user)
- PasswordResetToken uses `@ManyToOne` to User (user may have multiple tokens, only latest valid)
- BlacklistedToken is standalone (no FK to User; stores token string + expiry for cleanup)

## Requirements

### Functional
- User entity must have an `email` field (unique, nullable for backward compat with existing users)
- PasswordResetToken must store: UUID token string, user reference, expiry date, used flag
- RefreshToken must store: UUID token string, user reference (unique), expiry date
- BlacklistedToken must store: JWT string (or hash), token type enum, expiry date
- All repositories must extend JpaRepository with necessary query methods

### Non-Functional
- Entity files must stay under 200 lines
- Follow existing naming: lowercase plural table names, PascalCase entity names
- Use `@Data` Lombok on new entities (matches existing style)

### Step 2: Update application.yml

Add mail configuration and refresh token expiry under the `spring:` section and `namnd:` section:

```yaml
spring:
  mail:
    host: smtp.gmail.com
    port: 587
    username: ${MAIL_USERNAME:your-email@gmail.com}
    password: ${MAIL_PASSWORD:your-app-password}
    properties:
      mail:
        smtp:
          auth: true
          starttls:
            enable: true

namnd:

    jwtRefreshExpiration: 604800000
```

### Step 5: Create PasswordResetToken.java entity

Create `src/main/java/com/namnd/springjwt/model/PasswordResetToken.java`:

```java
package com.namnd.springjwt.model;

import lombok.Data;

import javax.persistence.*;
import java.util.Date;

@Data
@Entity
@Table(name = "password_reset_tokens")
public class PasswordResetToken {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false, unique = true)
    private String token;

    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "user_id", nullable = false)
    private User user;

    @Column(name = "expiry_date", nullable = false)
    @Temporal(TemporalType.TIMESTAMP)
    private Date expiryDate;

    @Column(nullable = false)
    private boolean used = false;
}
```

```java
package com.namnd.springjwt.model;

import lombok.Data;

import javax.persistence.*;
import java.util.Date;

@Data
@Entity
@Table(name = "refresh_tokens")
public class RefreshToken {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)

    @Column(nullable = false, unique = true)

    @OneToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "user_id", nullable = false, unique = true)

    @Temporal(TemporalType.TIMESTAMP)
```

```java
package com.namnd.springjwt.model;

import lombok.Data;

import javax.persistence.*;
import java.util.Date;

@Data
@Entity
@Table(name = "blacklisted_tokens")
public class BlacklistedToken {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false, unique = true)
    private String token;

    @Enumerated(EnumType.STRING)
    @Column(name = "token_type", nullable = false)
    private TokenType tokenType;

    @Column(name = "expiry_date", nullable = false)
    @Temporal(TemporalType.TIMESTAMP)
    private Date expiryDate;
```

## Risk Assessment

| Risk | Impact | Mitigation |
|------|--------|------------|
| `@Data` + `@Entity` with relationships (Lombok hashCode) | Medium - potential infinite recursion | PasswordResetToken uses LAZY fetch; RefreshToken's OneToOne is by user, not circular |
| `create-drop` DDL drops all data on restart | Low (dev only) | Acceptable for development; document for prod migration |
| Email column nullable allows users without email | Low | Intentional for backward compat; forgot-password simply won't work for those users |
| `deleteByUser` on RefreshTokenRepository needs @Transactional | Medium | Service layer method calling it must be annotated with @Transactional |

## Security Considerations

- BlacklistedToken stores the full JWT string; acceptable since JWTs are signed, not secret
- Password reset tokens are UUIDs (128-bit entropy), cryptographically secure via `UUID.randomUUID()`
- Refresh tokens are also UUIDs, opaque and stored in DB (not JWTs)
- Email field exposed only in registration; never returned in login response
- Mail credentials use environment variable placeholders, never hardcoded
